# EC-DR UX 規劃 v1（草稿）

目標：**剪輯用得順** ＋ **老闆一眼看懂進度**。
兩個角色各一頁主畫面，數字都可點進清單。

---

## A. 名詞定義（先對齊，避免做白工）

| 名詞 | 建議定義 | 對應資料 |
|---|---|---|
| 片源 / 新拍好的片 | 進來還沒剪的素材 | `stage = 待處理` |
| 剪輯中 | 已認領、製作中 | `stage = 剪輯中` |
| 已完成（待上片） | 剪好但還沒上 | `stage = 已完成` |
| 已上片 | 已發佈 | `stage = 已上片` |
| 新片 | 尚未上片（待處理＋剪輯中＋已完成） | 前三個 stage |
| 舊片 | 已上片的存檔 | `stage = 已上片` |
| 本週新拍 | 本週新增的片源 | **需新增 `createdAt` 欄位**（目前沒有）|
| 寵粉缺口 | 目標數 − 已完成/已排 | `mainType=寵粉`，目標見設定 |

> ⚠ 需要一個小資料補充：新增片源時寫入 `createdAt`，才能算「本週新拍好幾支、剪了幾支」。沒有它就只能顯示「目前各 stage 數量」。

- 「寵粉還差幾支」→ 類型達成卡的寵粉列，直接寫「還差 N 支」

## C. 剪輯工作台（重新設計，手機優先、少點擊）

```
✂️ 我的工作台（小明）                    語言：中文 ▸

┌──────────────────────────────────────┐
│   再剪 1 支就達標 💪                    │   ← 大字、激勵語
│   ●●○   今日完成 2 / 3                  │
└──────────────────────────────────────┘

進行中（1/3）
┌──────────────────────────────────────┐
│ 劉亦菲珠寶比較          [帶貨]          │
│ 已做 3 小時                            │
│   [   完成 ✔   ]        [ 求支援 ]     │
└──────────────────────────────────────┘

可認領的片源（4）                  ＋ 新增片源
┌──────────────────────────────────────┐
│ 招財貔貅開箱     [帶貨] 老闆自拍        │
│                          [  我來做  ]  │
├──────────────────────────────────────┤
│ 名人珠寶話題     [流量] 外部公司        │
│                          [  我來做  ]  │
└──────────────────────────────────────┘

📝 今日特別工作（選填）              ▸ 展開
```

**剪輯端順手原則：**
1. 最上方一張大卡只講一件事：**今天還差幾支**（激勵口吻），取代目前一排小數字。
2. 卡片式列表（取代表格），手機上單欄、按鈕大、一列一個主要動作。
3. 片源清單**自動排序**：今天缺的類型（例：寵粉/帶貨還沒達標）排最前，其次最舊的片源先剪。
4. 「今日特別工作」預設收合，沒事不干擾。
5. 「完成」彈窗維持「日期＋備份連結＋社群預排連結」兩連結即可（已是現況）。
