## A. Meta（IG 商業帳號 + FB 粉專）

你有 5 個 IG、1 個 FB。IG 要能抓「貼文洞察（觀看/互動）」必須是**商業或創作者帳號**，
而且**連到一個 FB 粉專**。抓數據走的是 **Instagram Graph API**。

1. 先確認每個 IG 都已切成「商業帳號」，並各自連到一個 FB 粉專
   （IG App → 設定 → 帳號類型與工具 → 切換為專業帳號 → 連結粉專）。
2. 到 <https://developers.facebook.com/> → 用你的 FB 登入 → 建立一個 App
   - 類型選「**商業（Business）**」。
   - App 名稱隨意（例：`ZanaGems Insights`）。
3. 在 App 裡加入產品：**Instagram Graph API**、**Facebook Login for Business**。
4. **商家驗證（Business Verification）**：App 設定 → 商家驗證，
   上傳公司登記等文件。這步會卡時間，先送。
5. **App Review**：申請以下權限（沒過就抓不到洞察）：
   - `instagram_basic`
   - `instagram_manage_insights`
   - `pages_read_engagement`
   - `pages_show_list`
   - （FB 粉專貼文洞察）`read_insights`
6. 送審時要錄一段「畫面操作說明影片」示範這些權限怎麼用——這段我可以幫你寫腳本。
7. 審核通過後，你會拿到 **App ID / App Secret**，以及各帳號的 **長效 Access Token**。
   → 這些**貼進 Supabase 的 Secrets**（B、C 之後我給你欄位名），不要貼進 GitHub、不要傳我。

> ⚠️ 老實說：Meta 這關（商家驗證＋App Review）通常是整個專案最久的一段，
> 可能要一兩週甚至更久。先送就對了。

## B. TikTok（5 個帳號）

TikTok 要抓自己帳號的貼文成效，走 **TikTok for Developers** 的
**Login Kit + Display API / Data Portability**。

1. 到 <https://developers.tiktok.com/> → 用其中一個 TikTok 登入 → 建立 App。
2. 加入 **Login Kit**，申請 scope：`user.info.basic`、`user.info.stats`、
   `video.list`（讀自己影片清單與統計）。
3. 送**審核**（TikTok 會問用途，如實填「管理自家帳號成效」）。
4. 5 個帳號都要各自「授權」給這個 App（OAuth 同意一次，拿到各自的 refresh token）。
5. 通過後拿到 **Client Key / Client Secret** 與各帳號 token → 一樣貼進 Supabase Secrets。

> ⚠️ 老實說：TikTok 官方 API 對「純數據回讀」的核准比較看用途，
> 有機會被要求補件或被拒。如果卡住，我們有備案（改用你每天手動貼一次數字、
> 或半自動匯入），但先照官方管道試。

## D. Firebase 服務帳號金鑰（**你自己產、你自己貼**）

Supabase 要能「寫回」我們的 Firestore，需要一把 Firebase 服務帳號金鑰。
**這把很敏感，等於資料庫的萬能鑰匙。由你產生、你貼進 Supabase Secrets，全程不要傳給我、不要進 GitHub。**

1. 到 <https://console.firebase.google.com/> → 選專案 **ec-dr-21416**。
2. 齒輪 ⚙️ → 專案設定 → **服務帳戶（Service accounts）** 分頁。
3. 按「**產生新的私密金鑰（Generate new private key）**」→ 下載一個 `.json` 檔。
4. **先別動它、別上傳任何地方。** 等我把 Supabase 後端寫好、告訴你要貼哪個欄位，
   你再把這個 `.json` 的內容整段貼進 Supabase 的 Secret（欄位名我會給，例如 `FIREBASE_SERVICE_ACCOUNT`）。
5. 貼完後，本機那個 `.json` 檔可以刪掉。

> 🔒 再強調：這一步我不經手。你只要在最後把它貼進 Supabase 就好。
