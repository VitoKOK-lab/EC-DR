# EC-DR UX 規劃 v2 — 全日工作流重構（草稿）

原則：**剪輯端＝快速輸入、快速產出、看懂自己的時間分配；老闆/HR端＝即時儀表板＋交辦。兩端介面不同。**

---

## 0. 一句話總結

> 剪輯一早「開工」→ 一路用最少點擊認領/產出/協助（全程自動計時）→ 下班一鍵匯報（系統已彙整）。
> 老闆一早打開就看到「每人幾點開工、現在在做什麼、完成幾支」，可對某人「📩 交辦」，對方 1 小時內快速回報，自動進當日匯報。

---

## 1. 需要新增的資料（最小集合）

| 集合/欄位 | 用途 |
|---|---|
| `videos.createdAt` | 片源何時進來（算「本週新拍」） |
| **`activity`**（新集合）`{user,date,ts,type,refId,label}` | 自動記錄每個動作 → 老闆時間軸、開工時間 |
| **`helpSessions`**（新集合）`{id,helper,target,videoId,startAt,endAt,minutes,note}` | 協助他人計時，工時歸戶給 helper |
| **`directives`**（新集合）`{id,from,to,text,createdAt,dueAt,status,reply,repliedAt}` | 老闆交辦＋1小時SLA＋回報 |
| `reports.autoSummary` | 下班匯報自動彙整內容（完成數/工時/協助/交辦回應） |

> 全部沿用現有 Firestore + onSnapshot 即時同步機制，不需後端改動。

## 2. 剪輯端 — 一天的動線

```
┌ 上班 ───────────────────────────────────────────────┐
│ 開 App → 一鍵「開工」(記 startAt，寫 activity)         │
│ 立刻看到：① 今日目標大卡 ② 老闆交辦收件匣(若有紅點)    │
└──────────────────────────────────────────────────────┘
        ↓
┌ 做事（全程自動計時）────────────────────────────────┐
│ ＋ 快速新增片源：只填「名稱＋類別」秒建，其餘後補      │
│ 認領「我來做」→ 自動 claimedAt 開始計時                │
│ 完成「✔」→ 自動算工時、計入今日產出                   │
│ 協助他人：在某人卡按「⏱ 開始協助」→ 結束 → 分鐘歸戶   │
│ 收到交辦：紅點 → 快速回報(範本/自訂) → 進匯報、停SLA   │
└──────────────────────────────────────────────────────┘
        ↓
┌ 下班 ───────────────────────────────────────────────┐
│ 「下班匯報」自動彙整：完成X支(各類型)、剪片工時、      │
│ 協助了誰共幾分鐘、特別工作、老闆交辦回應 → 一鍵送出   │
└──────────────────────────────────────────────────────┘
```

└──────────────────────────────────────┘

可認領（4）  ▸ 依今日缺口排序（寵粉/帶貨優先）
┌──────────────────────────────────────┐
│ 招財貔貅開箱   [帶貨] 老闆自拍          │
│                          [  我來做  ]  │
└──────────────────────────────────────┘

⏱ 協助別人  [ 選同事 ▾ ] [ 開始協助 ]    ← 計時歸自己

**要點**
- 開工一鍵、收件匣置頂、KPI 大卡只講「還差幾支」。
- 快速新增＝最少欄位；快速回報＝範本拉選；協助＝碼錶一鍵。
- 自己看得到「今日工時／協助佔多少」→ 滿足「知道自己的時間分配 & 幫別人花多少自己時間」。

### 3d. 其餘儀表板區塊（沿用 v1）
- 製作流水線（待剪→剪輯中→已完成→已上片）＋「本週新拍 X 支剪了幾支」

- 預排天數 / 落後人數 / 需支援
- 月排程行事曆

## 4. 老闆交辦 → 1 小時回報 → 自動進匯報（流程）

```
老闆送出 directive(to=小強, dueAt=+1h, status=待回報)
   → 小強端置頂紅點＋倒數
   → 小強點 [✅已完成/⏳進行中/✍自訂] 快速回報
   → directive.status=已回報, reply, repliedAt
   → 自動 append 到 reports[小強__今天].autoSummary「交辦回應」區
逾 1 小時未回報：該人卡片標紅「⏳逾時未回報」，老闆端可見
（不強制阻擋操作，只提醒/標記）
```

## 6. 分階段實作

**P1（介面，不大改資料，當天可上）**
- 剪輯工作台卡片化、KPI大卡、片源智慧排序、特別工作收合
- 老闆儀表板：流水線＋類型達成(寵粉缺口)＋健康指標

**P2（時間/時間軸）**
- `videos.createdAt`；自動 `activity` 記錄；剪輯「我的時間分配」；老闆「今日即時狀態列＋時間軸」

**P3（協助計時）**
- `helpSessions` 碼錶；工時歸戶；匯報納入協助時數

**P4（交辦＋SLA）**
- `directives` 交辦/快速回報/1小時倒數/自動進匯報/逾時標記

---

## 7. 決策（已定 ✅）
1. **開工時間**：一鍵「開工」打卡（記 `reports.startAt`）。
2. **協助計時**：事後填「幫了誰、幾分鐘」（不做碼錶）。
3. **逾時 1 小時**：標紅 ＋ 瀏覽器推播通知（Notification API，不強制阻擋）。
4. **主要裝置（重要，與初稿相反）**：**剪輯＝桌機優先**、**老闆/HR＝手機優先**。
   - 剪輯桌機：可用較寬版面、表格＋鍵盤快速輸入、批次操作。
   - 老闆手機：即時狀態列改為單欄堆疊、卡片大、交辦一鍵；行事曆改精簡。

## 8. 實作順序（依決策調整）
- **P1 開工打卡 ＋ 老闆「今日即時狀態列」**（幾點開工／現在在做什麼／完成幾支）— 多數可由現有資料推導，僅新增 `reports.startAt`。
- **P2 老闆手機版儀表板**＋剪輯桌機版工作台視覺重構。
- **P3 協助事後填報**（幫了誰幾分鐘）＋剪輯「我的時間分配」。
- **P4 交辦 `directives`**：快速拉選＋範本＋1 小時倒數＋自動進匯報＋逾時標紅＋推播。
